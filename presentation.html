02 · Concept

Vision du projet

Face aux trackers classiques qui affichent une simple liste d'événements, l'idée est née de combiner le suivi de colis avec la visualisation spatiale en temps réel — inspiré de conflicty.app et de FlightRadar24.

17TRACK API
3 200+ transporteurs
Événements de suivi
OpenSky / AviationStack
Vols live
Fallback simulé
Globe 3D
globe.gl + Three.js
Arcs animés

04 · Design

Pourquoi ce thème ?

Pas de wireframe traditionnel — globe.gl a structuré l'UI dès la découverte de la lib. Le globe est l'interface, pas la décoration.

Layout
  • → Globe = élément central (80–90 % de l'écran)
  • → Sidebar ~300px · collapsible mobile
  • → Modal contextuel déclenché au clic
Design System
  • → Palette orange / anthracite (mode sombre par défaut)
  • → Glassmorphism + bloom Three.js (intensité 1.6)
  • → Inter (UI) · JetBrains Mono (données)

06 · Fonctionnalités

Géocodage & Résolution GPS

Chaîne de fallback pour résoudre chaque position en coordonnées GPS précises — du plus fiable au dernier recours.

1. last_live_lat/lng Dernière position live OpenSky (vol aérien identifié)
2. Événement géocodé geopy · base statique de centroïdes
3. Centroïde pays destination Fallback pays dest (dest_coords)
4. Centroïde pays origine Dernier recours (origin_coords)
geocoding.py
07 · Fonctionnalités

Extraction de vol & SimulationEngine

✈️ Flight Extractor
  • → Regex [A-Z]{2,3}\d{3,4} sur descriptions 17TRACK
  • → Fallback AviationStack API
  • → Mapping carrier → airline cargo IATA
  • UPS→5X · FedEx→FX · DHL→D0
  • → Stocke flight_iata sur TrackingEvent
flight_extractor.py
⚙️ SimulationEngine
  • → Détection mode (air / road / sea / unknown)
  • → Calcul Haversine multi-segments
  • → estimated_departure / estimated_arrival
  • → Flag simulated: bool
  • → progress(t) → slerp dans Globe.tsx
simulation_engine.py
